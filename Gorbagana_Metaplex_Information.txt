Gorbagana: A Solana Fork Overview and Metaplex NFT Support

Solana‚Äôs Metaplex Protocol and NFTs

On Solana, the Metaplex protocol is the de facto standard for creating and managing NFTs (non-fungible tokens) and other digital assets. In fact, Metaplex‚Äôs programs and standards have been used to create virtually all tokens and NFTs on Solana. Much like ERC-721 (for NFTs) and ERC-20 (for fungible tokens) on Ethereum, the Metaplex Digital Asset Standard defines how NFTs work on Solana.

Key components of Metaplex on Solana include:

Token Metadata Program ‚Äì This is an on-chain program (part of the Metaplex Program Library) that attaches metadata to tokens (especially NFTs). It allows each NFT to have a name, symbol, description, image URI, attributes, etc., stored in a standardized way off-chain and referenced on-chain. This metadata ensures that wallets and explorers can display the token with its proper name, image, and attributes, rather than just showing a raw token address. In other words, thanks to Metaplex metadata, your Solana NFT shows up as ‚ÄúCool NFT #1 üé®‚Äù with an image, instead of an obscure token mint ID.

Candy Machine ‚Äì This is a popular Metaplex smart contract used for launching NFT collections on Solana. The Candy Machine program manages fair minting of a collection (often used to handle random distribution of minting, cap the supply, enforce mint dates, etc.). It has been the leading minting/distribution program for NFT drops on Solana. Many Solana NFT creators prepare their art and metadata, then use the Candy Machine CLI/tool to upload assets and handle the minting process for thousands of NFTs in a collection.


In simpler terms, Metaplex provides the infrastructure that makes Solana NFTs possible and easy to use ‚Äì it‚Äôs the reason Solana NFTs have standardized metadata and tooling. If you already have a collection ready for Solana, you likely have JSON metadata files for each NFT and perhaps planned to use a tool like Metaplex‚Äôs Candy Machine (or a derivative like Sugar CLI) to deploy the collection.

Metaplex Fork on Gorbagana (NFT Support on Gorbagana)

When you hear that ‚ÄúMetaplex was also forked for Gorbagana,‚Äù it means that the Gorbagana developers have ported or re-deployed the same NFT programs and standards from Solana‚Äôs Metaplex onto the Gorbagana blockchain. In fact, members of the Gorbagana team have reported that they successfully forked the Metaplex token metadata program and got it working on Gorbagana‚Äôs testnet. This was a crucial step because without a metadata program, tokens on Gorbagana would just appear as anonymous mints. With the Metaplex metadata in place, Gorbagana NFTs can have names, images, and attributes, and will display properly in any wallet or explorer that is configured for the Gorbagana network. In other words, you can set up an NFT mint on Gorbagana in a similar fashion as on Solana, with randomized minting, etc., using the familiar Metaplex workflow.

Bottom line: Gorbagana‚Äôs NFT ecosystem is intended to mirror Solana‚Äôs. If you deploy NFTs on Gorbagana, they will follow the same format and standards as Solana NFTs. Deploying Your NFT Collection to Gorbagana

Since Gorbagana is a fork of Solana with added support for Metaplex, deploying an NFT collection to Gorbagana will be very similar to deploying on Solana, with a few configuration changes:

1. Configure to use Gorbagana‚Äôs Network: You will need to point your tools (Solana CLI, Metaplex candy machine tool, or SDK scripts) to the Gorbagana network instead of Solana. Gorbagana provides a public RPC endpoint (https://rpc.gorbagana.wtf/) for its testnet that you can use in place of Solana‚Äôs RPC. For example, you can set your Solana CLI config to use that URL as the --url, or in code use that RPC URL. The Gorbagana docs note that while a custom Gorbagana CLI is in development, developers can use the standard Solana CLI with the Gorbagana RPC to interact with the chain.


2. Obtain $GOR for Transaction Fees: Just as you‚Äôd need SOL to pay for transactions on Solana, you‚Äôll need some GOR tokens to pay for transactions on Gorbagana. On the testnet, these are freely available from the Gorbagana faucet. Make sure your wallet on Gorbagana (likely a test wallet) has enough GOR to cover the cost of minting your NFTs and setting up a candy machine (fees are minimal, but each NFT mint and metadata upload will consume a small amount of GOR).


3. Use Metaplex Tools (Candy Machine or SDK): You can then use the same procedure you would on Solana to deploy your NFT collection. If you were planning to use the Metaplex Candy Machine CLI (or a tool like sugar CLI) on Solana, you can use it for Gorbagana by configuring it to connect to the Gorbagana RPC or adding Gorbagana as a custom network. The programs on Gorbagana responsible for NFTs are compatible, so the upload and mint commands should work similarly. Essentially, you will be creating the candy machine (or whatever minting mechanism you use) on the Gorbagana chain instead of Solana. All your NFT assets (image files, JSON metadata) can remain the same ‚Äì the only difference is the target network. The metadata JSON structure doesn‚Äôt change at all, since it‚Äôs the Metaplex standard on both chains.


4. Mint and Verify NFTs: Once you deploy (mint) the NFTs on Gorbagana, you‚Äôll want to verify everything. Since mainstream Solana block explorers or wallets won‚Äôt recognize the Gorbagana chain, you‚Äôll use Gorbagana-specific tools. For instance, you can use Trashscan (the Gorbagana block explorer) to look up your minted NFTs and accounts. For wallet support, currently Backpack wallet is the primary option to directly view and manage Gorbagana assets. You can add the Gorbagana RPC in Backpack‚Äôs network settings (as a custom Solana network) and then you'll be able to see your Gorbagana NFTs in that wallet interface. (Popular Solana wallets like Phantom or Solflare do not yet support custom networks like Gorbagana, so Backpack is recommended for now.)


5. Future Considerations: Keep in mind that Gorbagana‚Äôs mainnet is still under development. If your goal is to launch on Gorbagana‚Äôs eventual mainnet, you might be deploying on testnet now and later migrating or redeploying on mainnet when it goes live. Also, if bridging between Solana and Gorbagana becomes available for NFTs (as planned), your collection could potentially take advantage of that ‚Äì for example, allowing users to swap their NFT from one chain to the other. This would rely on both chains having the same metadata standard (which they do, via the Metaplex fork).



In summary, Metaplex being forked for Gorbagana means you don‚Äôt have to fundamentally rework your NFT collection to use this new chain. To deploy on Gorbagana, you‚Äôll mainly adjust network endpoints and use GOR tokens, but your overall workflow (preparing metadata, using candy machine or other minting scripts) remains largely the same as it would on Solana.
